# Data Connector — أيهم

## الهدف
توصيل كل مصادر البيانات والتجارب بالمشروع، وربط الجلسة، الذاكرة الدائمة، والتخزين الخارجي بالأرشيف الرمزي.

## الوحدات

### 1. Session Memory — ذاكرة الجلسة
- تخزين مؤقت لكل البيانات أثناء الجلسة.
- جميع التجارب الحالية تُسجّل هنا أولاً.
- متغيرات مرتبطة:
  - `session_id`
  - `timestamp`
  - `current_data`

### 2. Persistent Memory — الذاكرة الدائمة
- حفظ طويل الأمد للمعلومات والتجارب.
- قابل للفهرسة والتحليل المستقبلي.
- متغيرات مرتبطة:
  - `persistent_id`
  - `recorded_data`
  - `metadata`

### 3. External Storage — التخزين الخارجي
- نسخة احتياطية قابلة للنقل والتحليل.
- يربط المشروع بالنسخ المشفرة خارج النظام.
- متغيرات مرتبطة:
  - `external_id`
  - `encrypted_backup`
  - `transfer_protocol`

## Workflow سريع
1. جمع البيانات → Session Memory
2. مراجعة وتحليل → Persistent Memory
3. أرشفة نسخ احتياطية → External Storage
4. تحديث **ARCHIVE** ورموز **SYMBOLIC TEMPLATES**
5. توقيع كل عملية → SIG : `AYMEN × AI`

## ملاحظات
- جميع الإدخالات تُشفّر تلقائيًا قبل التخزين الخارجي.
- يمكن ربط أي مصدر بيانات جديد بنفس الهيكلية.
- كل تجربة تتحول إلى وحدة معرفة قابلة للفهرسة والتحليل.
